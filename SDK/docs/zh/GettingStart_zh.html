    # 从运行View开始

    # 1. OpenNCC相机运行权限

    进入CDK目录：Tools/deployment下，运行脚本“install_NCC_udev_rules.sh”，在终端下输入：
    ./install_NCC_udev_rules.sh
    获取相机挂载权限,成功后需要重启电脑。

    # 2. 解压OpenNCC View软件包，进入目录下，打开终端，输入：

    Ubuntu下：./AppRun，启动软件。（如果无法运行，输入:sudo ./AppRun再次尝试,并且确保Configuration目录下的moviUsbBoot文件允许作为程序可执行文件：右键属性-权限-允许作为程序可执行文件,也可以用命令sudo chmod +x moviUsbBoot）
    OpenNCC View软件成功启动后，接下来可以操作实现相机和View之间的交互，操作步骤：
    * 把OpenNCC相机连接到电脑usb3.0接口，点击 Get device info 按钮获取设备信息，此时log区域会显示设备信息(可以看下图中Log区域，提示USB interface is 3.0,yuv out flow is currently available)，如果提示连接不是3.0，请把OpenNCC相机Type-C接口连接的数据线旋转180°再次插入，点击Get device info 按钮获取设备信息。确保log显示为:USB interface is 3.0 ，如果显示不是3.0也能正常使用，但传输速率会下降,实际运行于USB2.0模式。
    * 任意选择一种出流格式 yuv420p/H.264/H.265/mjpeg，分辨率默认为1080p。
    * “Stream Resolution”：可以选择分辨率，有两种分辨率可供选择，1080p和4K。
    * 点击加载模型 “1st network model”：选择算法模型，目前支持10多种模型可选，选择None即不加载模型，仅出流显示视频，而选择加载算法模型后，可以通过框选ROI区域，框选出区域，算法只对区域内的场景进行识别。
    * “2nd network model”：可以参考下面的2.1二级模型演示。
    * “Model Score” 是算法识别的最低分数，达到后算法才会在画面中框选出物体。
    * “Display Scaler” 是视频显示窗体大小，可以调节显示窗口分辨率。
    * “Exposure control”：设置曝光，可以选择Auto和Manual，当选择Manual时，可以自己设置times和iso。
    * 勾选“show state”，选择是否在画面上显示当前状态信息，包括视频流帧率、算法帧率、分辨率、设备id。
    * 勾选“inference accelerate”，选择是否启用算法加速.（必须在加载算法模型前选择）
    * 勾选“save_avi”，会在打开视频流后，将视频保存到avi文件夹下，保存文件的名称是以时间保存，当关闭视频流后，视频会保存下来，yuv420p没有此项。
    * 点击”Start running models”，加载算法模型，成功打开视频流。

    • 在“1st network model”下拉菜单中选择vehicle-license-plate-detection-barrier-0106-fp16.blob
    • 选择后，2nd network model下可选择需要的模型，目前演示只支持一个二级模型(车牌识别)。用户需要可以自行添加。

    # 从使用SDK开始

    1. 下载OpenNCC开发包,并需要安装依赖包。
    2. 进入Samples/How_to/Load a model
    3. make clean, make
    4. cd ../../bin/
    5. 将OpenNCC相机和主机连接，./Openncc运行
